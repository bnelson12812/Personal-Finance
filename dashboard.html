category rollup structure

needs (50% of income)
• housing — rent/mortgage
• groceries — food shopping
• transportation — gas, train, car expenses
• utilities — electric, water, internet
wants (30% of income)
• dining — restaurants, delivery
• shopping — retail, online purchases
• entertainment — movies, events, subscriptions
• health & fitness — gym, wellness
• pharmacy — prescriptions, health items
• business services — professional fees
savings (20% of income)
calculated as income minus needs minus wants

how merchant grouping works

transactions are automatically grouped by merchant name for easier tracking:
• numbers and special characters are stripped
• first 3 words of description are used
• similar merchants cluster together
examples:
shell #1234 → groups as "shell"
shell #5678 → groups as "shell"
trader joe's #421 → groups as "trader joe's"
trader joe's #305 → groups as "trader joe's"
amazon.com*abc123 → groups as "amazon com"
amazon.com*xyz789 → groups as "amazon com"

monthly file upload process

upload fresh data each month around the 6th (after your paycheck hits):
step 1: export from your bank
• checking account → save as Debit[MMYYYY].csv
• credit card → save as Credit[MMYYYY].csv
• hysa → save as HYSA[MMYYYY].csv (full history)
• fidelity roth → save as Roth[MMYYYY].csv (positions)
step 2: upload to github
• go to your github repo → data/bnelson/
• click "add file" → "upload files"
• drag your 4 csv files
• commit changes
step 3: refresh dashboard
• return to dashboard
• click ↺ refresh in top banner
• new month appears automatically
important: avoid duplicate data
when exporting from your bank, only export the current month to prevent duplicate transactions. if you accidentally upload overlapping data, re-export and replace the files.
